# macros

## Rust procedural macros

Operate at the token level (AST). I.e. a macro takes in a sequence of code as input, like a function, struct, etc., and produces a new sequence of code as output. They are used to generate code at compile time, allowing for powerful code generation and metaprogramming capabilities.

`cargo-expand` is a tool that allows you to see the expanded code generated by procedural macros, which can be helpful for debugging and understanding how the macros work.

`cargo-expand` relies on a `nightly` compiler, as it uses unstable features of the Rust compiler to perform the code expansion. This means that you need to have a nightly version of Rust installed to use `cargo-expand`.

```bash
rustup toolchain install nightly --allow-downgrade # --allow-downgrade instructs rustup to find the latest nightly build that has all the toolchain components that the one currently installed
rustup default nightly
cargo install cargo-expand
cargo expand

# OR

# Use the nightly toolchain just for this command invocation
cargo +nightly expand
```

```bash
# Install nightly with specific components, allowing downgrade if needed
rustup toolchain install nightly \
  --component rust-src \
  --component llvm-tools-preview \
  --component clippy \
  --allow-downgrade
```

**check if this is still the case** successfully ran `cargo expand` with cargo-expand 1.0.121 and stable rustc 1.92.0 (ded5c06cf 2025-12-08)

Expanding a specific test file

```bash
cargo expand --test health_check
```
